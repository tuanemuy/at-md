# E2Eテスト

このディレクトリには、AT-MDアプリケーションのエンドツーエンド（E2E）テストが含まれています。

## テストの概要

E2Eテストは、アプリケーション全体が実際のユーザーの視点から正しく動作することを確認するためのテストです。このテストでは、以下の項目をテストします：

1. **アプリケーションフロー**：ユーザーがアプリケーションを使用する一連のフローをテストします。
   - ホームページの表示
   - コンテンツ詳細ページへの遷移
   - ユーザー詳細ページへの遷移
   - フィード詳細ページへの遷移

2. **API**：REST APIとGraphQL APIの両方をテストします。
   - REST APIのエンドポイントテスト
   - GraphQL APIのクエリとミューテーションテスト

## テストの実行方法

### 前提条件

E2Eテストを実行する前に、以下の準備が必要です：

1. アプリケーションサーバーが起動していること
2. データベースが正しく設定されていること

### テストコマンド

```bash
# すべてのE2Eテストを実行
deno test --allow-all tests/e2e

# 特定のE2Eテストを実行
deno test --allow-all tests/e2e/app-flow.test.ts
deno test --allow-all tests/e2e/api-flow.test.ts
```

## テスト実行時の注意点

1. **環境変数**：テスト実行時に必要な環境変数が設定されていることを確認してください。

2. **APIテスト**：APIテストを実行する場合は、APIサーバーが起動していることを確認してください。

3. **データベース**：テスト実行前にテスト用のデータベースが正しく設定されていることを確認してください。テストによってはデータベースの内容が変更される可能性があります。

4. **モックデータ**：テストで使用するモックデータは `setup.ts` ファイルで定義されています。必要に応じて修正してください。

5. **ブラウザ環境**：テストはJSDOM（JavaScript実装のDOM）を使用してブラウザ環境をエミュレートしています。実際のブラウザとは挙動が異なる場合があります。

## テストファイルの説明

- `setup.ts`：テスト環境のセットアップを行うファイル
- `app-flow.test.ts`：アプリケーションフローのテスト
- `api-flow.test.ts`：APIのテスト

## テスト結果の確認

テスト実行後、成功したテストと失敗したテストの数が表示されます。失敗したテストがある場合は、エラーメッセージを確認して問題を修正してください。

## テストの追加方法

新しいE2Eテストを追加する場合は、以下の手順に従ってください：

1. 新しいテストファイルを作成（例：`new-feature.test.ts`）
2. 必要なモジュールをインポート
3. テストケースを記述
4. テストを実行して動作確認

## トラブルシューティング

- **テストが失敗する場合**：エラーメッセージを確認し、問題を修正してください。
- **環境変数の問題**：`.env`ファイルが正しく設定されていることを確認してください。
- **APIサーバーの問題**：APIサーバーが起動していることを確認してください。
- **データベースの問題**：データベースが正しく設定されていることを確認してください。
